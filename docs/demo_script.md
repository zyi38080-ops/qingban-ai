# 晴伴AI 课堂演示脚本（3-5分钟）

## 演示准备

- 确认 GitHub Pages 页面可正常打开
- 确认 Demo Mode 正常工作（或已接入真实 Qwen）
- 准备好演示用输入文本

---

## 演示步骤

### 第1步：打开页面（15秒）

> 打开浏览器，访问晴伴AI GitHub Pages 页面。

**讲解词**："这是晴伴AI，一个面向青少年游戏行为失控的隐私陪伴与行为支持智能体。"

### 第2步：介绍项目定位（20秒）

**讲解词**："晴伴AI不是普通聊天机器人，它专门面向游戏时间失控、冲动充值、熬夜玩游戏、与父母因游戏发生冲突等场景。系统不提供医疗诊断，页面顶部有明确的免责声明。"

👉 指向页面顶部的"隐私可控的游戏行为陪伴与支持智能体"和免责声明。

### 第3步：演示隐私脱敏（30秒）

> 在聊天输入框中输入：
> "我叫张三，手机号是13812345678，我昨晚又充了500元。"

**讲解词**："注意观察，当我输入包含手机号的文本时，系统会——"

👉 发送后，指向脱敏提示：

**讲解词**："自动识别并隐藏手机号，显示'系统已自动隐藏可能的敏感信息：手机号'。这是一个前端实现的隐私脱敏功能。"

### 第4步：演示陪聊功能（45秒）

> 输入："我晚上总是玩游戏到两点。"

**讲解词**："当用户表达游戏熬夜的问题时，晴伴会按照【回应】【建议】【下一步】三段式格式回复。同时会自动匹配知识库，并在下方展示行动计划卡。"

👉 指向回复中的三段格式、行动计划卡和知识库建议。

### 第5步：演示知识库和行动计划（20秒）

**讲解词**："左下角的参考建议来自本地知识库，包含12条高质量知识片段。行动计划卡是基于规则生成的，内容简短、具体、可执行。"

👉 展示知识库卡片和行动计划卡片。

### 第6步：演示隐私设置（25秒）

> 点击左侧"隐私"导航按钮。

**讲解词**："隐私设置面板允许用户设置昵称、回复风格、是否授权保存数据。默认状态下不保存聊天内容、不保存身份信息。用户可以随时点击'清空本地数据'按钮删除所有本地存储。"

👉 展示清空数据功能。

### 第7步：演示高风险安全分流（20秒）

> 点击"陪聊"回到聊天，输入："我觉得活着没意思了"

**讲解词**："当用户输入高风险内容时，系统不会调用普通聊天流程，而是直接显示安全提示，建议用户联系可信任的人或紧急服务。"

👉 展示红色安全提示。

### 第8步：展示系统工作流（15秒）

> 点击左侧"工作流"导航按钮。

**讲解词**："这是系统的完整工作流程：用户输入→隐私脱敏→风险识别→知识库检索→Qwen3.7-plus生成回答→行动计划生成→用户授权后保存偏好。LLM使用阿里云百炼Qwen3.7-plus，通过Cloudflare Worker作为API代理保护密钥。"

👉 展示流程图和技术栈说明。

### 第9步：总结（10秒）

**讲解词**："晴伴AI体现了智能体的四个核心组成：LLM大语言模型、工具调用、知识库和工作流。同时展示了隐私人格设定和隐私数据处理两个课程重点。谢谢！"

---

## 常见课堂提问预案

| 问题 | 回答 |
|------|------|
| 为什么用Qwen不用ChatGPT？ | 课程要求使用阿里云百炼平台，Qwen3.7-plus性能足够且成本低 |
| 数据存在哪里？ | 隐私设计优先，默认不保存聊天内容；用户主动授权后仅在浏览器LocalStorage中保存偏好 |
| 如何处理高风险用户？ | 关键词规则识别→安全分流→固定安全提示，不调用LLM |
| 为什么不用向量数据库？ | 课程要求轻量实现，12条知识片段用关键词匹配已足够 |
| 怎么保护API Key？ | 前端不直接调用阿里云API，通过Cloudflare Worker代理，Key存在Worker环境变量中 |
